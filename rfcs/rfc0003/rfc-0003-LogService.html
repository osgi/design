Java Log Service for OSGi Framework (Log Service)

Status of this Memo

This document specifies a Java Log Service for the OSGi Framework (Log Service), and requests discussion and suggestions for improvements. Distribution of this document is unlimited.

Abstract

A key component of the OSGi Framework[1] is a Log Service which will provide a Java API to allow Java programs running in the Framework to write information to the log and also to read entries from the log. This RFC describes an OSGi Service API that supports the requirements in Java Log Service RFP 0003[2].

Acknowledgment

This specification is based upon prior work of members of the Connected Alliance and The Open Services Gateway Initiative (OSGi).

Introduction

The Log Service API defines the set of capabilities to allow Java programs written to run in the OSGi Framework to write entries to the log and for entries to be read from the log.

Motivation and Rational

For diagnostic and reliability purposes, Java programs written to run on the OSGi Framework will need to write information to a log that can be examined, if necessary, for problem determination. This LogService API will provide such a capability. In addition to the ability to write to the log, the capability to read the log must also exist. The companion LogReaderService API will provide this support.

Technical Discussion

The LogService provides the following capabilities and features:

·        Records the current system time in a log entry when created.

·        Records a severity level in a log entry when created.

·        Records a text message in a log entry when created.

·        Records an optional Java Throwable object in a log entry when created.

·        Records the identity (BundleContext) of the Java program running in the Framework that called LogService to create the log entry.

·        Listens to Framework events and create log entries representing the events.

 

The LogReaderService provides the following capabilities and features:

·        Read past log entries via an enumeration.

·        Subscribe and unsubscribe using a listener to log entries as they are created.

 

The API for the Log Service is described in the Java source files and Javadoc HTML which are included as part of this RFC (rfc-0003.zip).

Security Considerations

In secure implementations of the OSGi Framework and Log Service, the ability to read entries from the Log should be protected. Thus ServicePermission for the LogReaderService should only be given to those bundles that should be allowed to read the log. In general, all bundles should have ServicePermission for the LogService.
